intern()

如果字符串常量池中没有对应的字符串的话，则在常量池中生成，并将引用地址改为常量池。如果存在就返回字符串，

保证变量s指向的是字符串常量池中的数据（另一种方式String s = “abc”）

    String s = new String("1");
    s.intern();  //常量池中已经存在"1"，s还是指向堆中的对象
    String s2 = "1";
    System.out.println(s == s2);  //jdk6:false  jdk7/jdk8:false

    String s3 = new String("1") + new String("1");
    s3.intern();  //常量池中不存在"11"，生成后s3指向常量池
    String s4 = "11";
    System.out.println(s3 == s4);  //jdk6:false  jdk7/jdk8:true

题目1：new String("ab")会创建几个对象
    两个对象，看字节码就知道了
    一个对象是：new关键字在堆空间中创建的
    另一个对象是：字符串常量池中的对象 
题目2：new String("a") + new String("b")会创建几个对象
   	对象1：new StringBuilder()
    对象2：new String()
    对象3：常量池中的"a"
    对象4：new String()
    对象5：常量池中的"b"
    StringBuilder的toString方法生成对象6
    对象6：new String("ab")
    toString()的调用，在字符串常量池中，没有生成"ab"
